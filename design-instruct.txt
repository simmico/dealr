# DEALR PAGE CREATION GUIDE

This document serves as the master instruction for creating new pages on the Dealr platform. Adhere to these patterns to ensure consistency across the codebase.

## 1. PROJECT ARCHITECTURE
Dealr is a "Vanilla-Plus" platform: Vanilla HTML5, CSS3, and ES6+ JavaScript. It uses a data-driven approach where JSON files in `/data/` drive the content rendered by JS modules in `/js/`.

## 2. FILE NAMING CONVENTIONS
- **HTML**: `kebab-case.html` (e.g., `rewards-travel.html`)
- **JS**: `kebab-case.js` in `/js/` (e.g., `js/travel.js`)
- **JSON**: `kebab-case.json` in `/data/` (e.g., `data/travel.json`)
- **CSS**: Use existing modular files. Add page-specific styles to `css/pages.css`.

## 3. HTML BOILERPLATE
Every page MUST include:
- **Head**: Standard meta tags, viewport, theme-color (#081B1B), and favicon/manifest links.
- **CSS Imports**:
  - `css/base.css` (Variables & Resets)
  - `css/layout.css` (Nav/Footer)
  - `css/components.css` (Buttons/Cards)
  - `css/animations.css` (Transitions)
  - `css/pages.css` (Page-specifics)
- **Body Class**: `page-{name}` (e.g., `page-banking`)
- **Navigation**: Desktop `.nav` and Mobile `.mobile-nav`. Update `.nav__link--active` and `.mobile-nav__tab--active`. The mobile experience is PWA-like with a fixed bottom bar.
- **Gold Thread**: The `<div class="gold-thread">` SVG on the left.
- **Bot Widget**: The `.bot-container` and its associated logic.
- **Scripts**: 
  - `js/scroll.js` (Navigation behavior)
  - `js/bot.js` (Assistant)
  - Page-specific module (e.g., `js/banking.js`)

## 4. CSS & STYLING (BEM)
Use BEM (Block Element Modifier) methodology.
- **Colors**: Use variables from `base.css` (`--color-mint`, `--color-gold`, `--color-dark`, `--color-beige`).
- **Containers**: Wrap content in `.container` for standard width.
- **Sections**: Use `.section` with modifiers like `.section--beige` or `.section--mid`.
- **Headers**: Standard `.section-header` with `.section-eyebrow`, `h2`, and `.section-subtitle`.

## 5. JAVASCRIPT & DATA PATTERN
New data-driven pages should follow the "Fetch-Process-Render" pattern:
1. **Load**: `fetch('/data/your-page.json')`.
2. **Process**: Filter, sort, or score the data (see `js/banking.js` for scoring logic).
3. **Render**: Build HTML strings and inject into the DOM.
4. **Animate**: Use `addAnimationClasses` from `js/utils.js` on elements with `.animate-ready`.

## 6. COMPONENTS
- **Buttons**: `.btn .btn--primary` or `.btn .btn--outline`.
- **Cards**: Use `.grid` (e.g., `.grid--3`) with cards that have `.animate-ready`.
- **Filter Bars**: Use `.filter-bar` with `.filter-btn`.

## 7. MOBILE & PWA EXPERIENCE
Dealr is designed to feel like a native app on mobile devices:
- **Bottom Navigation**: The `.mobile-nav` is a fixed bottom bar with tabs for primary sections (Home, Deals, Eat, Rewards).
- **More Drawer**: Secondary navigation items live in the "More" drawer, which slides up from the bottom.
- **PWA Manifest**: The `manifest.json` configures the app to run in `standalone` mode with a themed status bar (`#081B1B`).
- **Icons**: SVG icons in `/icons/` are used for the manifest and home screen shortcuts (192x192 and 512x512).
- **Safe Areas**: Use `env(safe-area-inset-bottom)` to ensure the bottom nav clears mobile home indicators.

## 8. CHECKLIST FOR NEW PAGES
- [ ] Create `{page}.html`.
- [ ] Create `data/{page}.json` (if data-driven).
- [ ] Create `js/{page}.js` (if interactive).
- [ ] Add the page to the desktop nav (`index.html` and all other pages).
- [ ] Add the page to the mobile bottom nav or "More" drawer.
- [ ] Add page-specific CSS to `css/pages.css`.
- [ ] Verify SEO meta tags and Open Graph properties.
- [ ] Ensure mobile bottom nav tabs are correctly activated.
